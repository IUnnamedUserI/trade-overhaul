# Пакетная торговля в Trade Overhaul

## Описание

В Trade Overhaul реализована система пакетной торговли, которая позволяет игрокам покупать и продавать товары партиями по фиксированному соотношению. Это улучшает экономическую систему игры, позволяя совершать более крупные сделки за одно действие.

## Как это работает

### Покупка у жителя
- Игрок платит 1 изумруд и получает определённое количество предметов (например, 10 морковок)
- Количество предметов за 1 изумруд задаётся в конфигурации как `buyQuantity`

### Продажа жителю
- Игрок продает определённое количество предметов (например, 8 морковок) за 1 изумруд
- Количество предметов за 1 изумруд задаётся в конфигурации как `sellQuantity`

## Конфигурация

В файлах профессий (.json) можно указать следующие параметры:

### Для конкретных предметов (staticPool):
```json
{
  "item": "minecraft:carrot",
  "buy": 1,
  "sell": 1,
  "buyQuantity": 15,      // Игрок получает 15 морковок за 1 изумруд
  "sellQuantity": 8,      // Игрок продает 8 морковок за 1 изумруд
  "minStock": 15,
  "maxStock": 45
}
```

### Для групп предметов (generalPool):
```json
{
  "tag": "minecraft:crops",
  "minStock": 12,
  "maxStock": 36,
  "buyPrice": 1,
  "sellPrice": 1,
  "buyQuantity": 12,      // За 1 изумруд игрок получает 12 любых культур
  "sellQuantity": 6       // За 1 изумруд игрок продает 6 любых культур
}
```

### Для покупки у игрока (buyPool):
```json
{
  "tag": "c:seeds",
  "buyPrice": 1,
  "buyQuantity": 20,      // Житель дает 1 изумруд за 20 семян
  "discountFactor": 70
}
```

## Особенности реализации

1. **Кратность запасов**: При обновлении инвентаря жителя количество предметов всегда кратно значению `buyQuantity`, чтобы избежать ситуаций, когда у жителя 13 морковок, а купить можно только по 5 штук.

2. **Гибкая торговля**: Игрок может использовать Shift+клик для покупки/продажи максимально возможного количества, кратного базовому соотношению.

3. **Экономическая балансировка**: Позволяет настроить цены так, чтобы крупные сделки были выгоднее, чем множество мелких.

## Примеры

- Фермер продаёт 15 морковок за 1 изумруд (`buyQuantity`: 15)
- Игрок продает 8 морковок фермеру за 1 изумруд (`sellQuantity`: 8)
- У фермера в инвентаре всегда будет 15, 30, 45 и т.д. морковок, но не 17 или 23

## Преимущества

- Упрощает крупные коммерческие операции
- Повышает реалистичность экономики
- Позволяет лучше контролировать цены
- Удобство для игроков при массовой торговле
