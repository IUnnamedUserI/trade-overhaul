# Руководство по конфигурации пакетной торговли

## Общие принципы

Система пакетной торговли позволяет настраивать соотношение предметов к изумрудам при покупке и продаже. Это улучшает экономическую модель игры, делая крупные сделки более удобными.

## Параметры пакетной торговли

### buyQuantity
- **Что означает**: Сколько предметов игрок получает за 1 изумруд при покупке у жителя
- **Пример**: `"buyQuantity": 10` означает, что за 1 изумруд игрок получит 10 единиц товара
- **Использование**: В разделах `staticPool`, `generalPool`

### sellQuantity  
- **Что означает**: Сколько предметов игрок должен продать, чтобы получить 1 изумруд
- **Пример**: `"sellQuantity": 8` означает, что за 8 предметов игрок получит 1 изумруд
- **Использование**: В разделах `staticPool`, `generalPool`, `buyPool`

## Примеры настройки

### Простой случай
```json
{
  "item": "minecraft:carrot",
  "buy": 1,
  "sell": 1,
  "buyQuantity": 15,      // Игрок покупает 15 морковок за 1 изумруд
  "sellQuantity": 8,      // Игрок продает 8 морковок за 1 изумруд
  "minStock": 15,         // Минимум в инвентаре жителя (кратно buyQuantity)
  "maxStock": 45          // Максимум в инвентаре жителя (кратно buyQuantity)
}
```

### Для групп предметов
```json
{
  "tag": "minecraft:wool",
  "minStock": 10,
  "maxStock": 30,
  "buyPrice": 1,
  "sellPrice": 1,
  "buyQuantity": 10,      // За 1 изумруд игрок получает 10 шерсти любого цвета
  "sellQuantity": 5       // За 1 изумруд игрок продает 5 шерсти любого цвета
}
```

### При покупке у игрока
```json
{
  "tag": "c:ores",
  "buyPrice": 2,          // Житель платит 2 изумруда за пакет
  "buyQuantity": 4,       // Житель покупает 4 руды за 2 изумруда (2 руды за 1 изумруд)
  "discountFactor": 60
}
```

## Важные замечания

1. **Кратность запасов**: Количество предметов в инвентаре жителя всегда будет кратно значению `buyQuantity` для обеспечения возможности покупки целых пакетов.

2. **Массовая торговля**: Используйте Shift+клик для покупки/продажи максимально возможного количества пакетов за раз.

3. **Балансировка**: Убедитесь, что `buyQuantity` и `sellQuantity` логически связаны с вашей экономической моделью. Обычно `buyQuantity` > `sellQuantity` для одного и того же товара, чтобы житель зарабатывал на перепродаже.

4. **Минимальные и максимальные запасы**: Значения `minStock` и `maxStock` должны быть кратны `buyQuantity` для корректной работы системы.

## Примеры балансировки

- **Агрессивная торговля** (выгодная для жителя): 
  - Покупка: 1 изумруд за 5 яблок (`buyQuantity`: 5)
  - Продажа: 8 яблок за 1 изумруд (`sellQuantity`: 8)
  - Житель покупает дешевле, продает дороже

- **Дружелюбная торговля** (выгодная для игрока):
  - Покупка: 1 изумруд за 10 яблок (`buyQuantity`: 10)  
  - Продажа: 6 яблок за 1 изумруд (`sellQuantity`: 6)
  - Игрок покупает дешевле, продает дороже
